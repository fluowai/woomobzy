# Dominios customizados com SSL automatico

## Arquitetura

O Imobzy usa Traefik como proxy de borda e o backend Node.js como controlador de dominios. Cada dominio de cliente gera um arquivo especifico em `/traefik/dynamic`; o file provider do Traefik observa essa pasta com `watch: true`, carrega a rota sem restart e emite o certificado via Let's Encrypt HTTP Challenge.

Componentes:

- `traefik/traefik.yml`: configuracao estatica do Traefik, entrypoints `web` e `websecure`, file provider e ACME.
- `traefik/dynamic/00-services.yml`: servicos compartilhados `imobzy_frontend` e `imobzy_api`.
- `traefik/dynamic/00-platform.yml`: rotas fixas dos dominios da plataforma.
- `server/domainService.js`: valida DNS, cria/remove configs dinamicas e verifica SSL.
- `docker-compose.yml`: Traefik, frontend, API e servicos auxiliares na mesma rede Docker.

## Fluxo automatico

1. Cliente informa `www.cliente.com.br` no painel.
2. API normaliza e valida o dominio.
3. API verifica se o registro A aponta para `PLATFORM_PUBLIC_IP`.
4. API rejeita duplicidade em `organizations.custom_domain` e `domains.domain`.
5. API grava o dominio no banco.
6. API escreve `/traefik/dynamic/www_cliente_com_br.yml`.
7. Traefik detecta o novo arquivo automaticamente.
8. Primeiro acesso HTTPS dispara emissao/uso do certificado Let's Encrypt.
9. O painel passa a exibir DNS e SSL separadamente.

## Exemplo de arquivo dinamico gerado

Para `www.okaimoveis.com.br`, o backend gera:

```yaml
# Generated by Imobzy domainService. Do not edit manually.
# Domain: www.okaimoveis.com.br

http:
  routers:
    www_okaimoveis_com_br_api:
      rule: "Host(`www.okaimoveis.com.br`) && PathPrefix(`/api`)"
      entryPoints:
        - websecure
      priority: 1000
      tls:
        certResolver: letsencryptresolver
      service: imobzy_api@file

    www_okaimoveis_com_br_frontend:
      rule: "Host(`www.okaimoveis.com.br`)"
      entryPoints:
        - websecure
      priority: 100
      tls:
        certResolver: letsencryptresolver
      service: imobzy_frontend@file
```

## Preparacao do servidor

No Linux de producao:

```bash
mkdir -p traefik/dynamic
touch traefik/acme.json
chmod 600 traefik/acme.json
docker compose up -d --build
```

Portas obrigatorias:

- `80/tcp`: HTTP Challenge do Let's Encrypt.
- `443/tcp`: trafego HTTPS.

## Portainer com Traefik em stack separada

Quando o Traefik nao roda dentro da mesma stack da aplicacao, ele precisa montar o mesmo volume dinamico usado pela API. Sem isso, o backend grava o arquivo do dominio, mas o Traefik continua servindo o certificado default.

Na stack da API:

```yaml
api:
  environment:
    TRAEFIK_DYNAMIC_DIR: /app/traefik/dynamic
    TRAEFIK_FRONTEND_SERVICE: imobfluow_frontend@docker
    TRAEFIK_API_SERVICE: imobfluow_api@docker
  volumes:
    - imobzy_traefik_dynamic:/app/traefik/dynamic

volumes:
  imobzy_traefik_dynamic:
    external: true
```

Na stack do Traefik, uma unica vez:

```yaml
services:
  traefik:
    volumes:
      - imobzy_traefik_dynamic:/traefik/dynamic:ro
    command:
      - "--providers.file.directory=/traefik/dynamic"
      - "--providers.file.watch=true"

volumes:
  imobzy_traefik_dynamic:
    external: true
```

Se o Traefik usa `traefik.yml` em vez de `command`, mantenha:

```yaml
providers:
  file:
    directory: /traefik/dynamic
    watch: true
```

## Boas praticas

- Nao use `HostRegexp({host:.+})` para dominios de clientes; gere uma rota `Host()` por dominio.
- Nao habilite HSTS agressivo para dominios recem-conectados antes de confirmar SSL ativo.
- Mantenha `traefik/acme.json` fora do Git; ele contem chaves privadas.
- Monitore logs do Traefik para `rate limit`, `authorization failed` e `too many certificates`.
- Para milhares de clientes, mantenha um arquivo por dominio. Isso reduz lock contention e facilita remocao atomica.
- Use DNS-01 apenas se precisar wildcard; para dominios customizados individuais, HTTP-01 e suficiente e mais simples.
